## рҹ”ӯ What is Code-Eclipse?

**Code-Eclipse** is a full-stack developer tool that parses raw Java source code and renders it as a live, interactive OOP tree diagram. No plugins. No setup. Just paste your Java code, hit **Parse**, and watch your entire class hierarchy come to life in seconds.

Whether you're a student learning OOP concepts, a developer reviewing an unfamiliar codebase, or a teacher explaining inheritance chains вҖ” Code-Eclipse makes it visual, beautiful and instant.

```
  Paste Java Code  вҶ’  Hit Parse  вҶ’  See Your OOP Tree
```

| Feature | Description |

| вҡЎ **Instant Parsing** | Paste Java code and get a live tree in milliseconds |
| рҹҢІ **Smart Tree Layout** | Auto-arranged with Dagre algorithm (top вҶ’ bottom hierarchy) |

## вҡҷпёҸ Getting Started

### Prerequisites

- **Node.js** v18 or higher
- **MongoDB** running locally or a MongoDB Atlas URI

---

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/code-eclipse.git
cd code-eclipse
```

---

### 2. Setup the Backend

```bash
cd backend
npm install
```

Create a `.env` file inside `/backend`:

```env
MONGO_URI=mongodb://localhost:27017/code-eclipse
PORT=5000
```

Add `"type": "module"` to `backend/package.json`:

```json
{
  "type": "module"
}
```

Start the backend server:

```bash
node server.js
```

Expected output:

```
вң… MongoDB connected
рҹҡҖ Server running at http://localhost:5000
```

---

### 3. Setup the Frontend

```bash
cd ../frontend
npm install
npm install dagre
npm run dev
```

---

### 4. Open in Browser

```
http://localhost:5173
```

## рҹҗӣ Known Limitations

- Parser uses **Regex** not a full AST вҖ” works great for standard Java OOP patterns
- Deeply nested anonymous classes or lambdas may not parse perfectly
- Generic types like `List<String>` are treated as plain types
